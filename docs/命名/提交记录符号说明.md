## 提交标题

| 图标  | 标签   | 说明                                 |
|-----|------|------------------------------------|
| 📤  | 新增   | 新增了一个**完整的**功能                     |
| ✏️  | 修改   | 大量修改结果的更新，**增删改**都算                |
| 🪛  | 重构   | 只**修改流程**，不大改结果的更新                 |
| 🗃️ | 杂项更新 | 一些和业务流程无关的更新（依赖配置等），或补充一些被漏的提交     |
| 💾  | 备份   | 不是提交完整的模块，而是整个项目做备份方便回滚（**常用于前端**） |
| 🚀  | 重大更新 | 如果因为**客观原因**（数据库崩溃、无法测试等）长期写了代码的补交 |

## 标注文件修改属性

| 图标  | 标签   | 说明                               |
|-----|------|----------------------------------|
| 📤  | 新增   | 这是一个新增的文件                        |
| ✏️  | 修改   | 这是一个修改的文件                        |
| 🪛  | 重构   | 这是一个重构的文件                        |
| 🗃️ | 杂项更新 | 这是一个补交的、或者业务流程无关的文件（依赖配置等）       |
| 🗑️ | 删除   | 这个文件被删除，**不会保留备份**，如果需要请检查历史提交记录 |
| 🐛  | 揥蚆蛒  | （戏称debug）对于错误点的微调                |
| 📁  | 文档   | 这个文件是一个文档                        |
| 🖇️ | 附带   | 这个文件没有流程的修改，但是被其他类带着附带修改了        |
