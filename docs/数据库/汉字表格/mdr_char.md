# 概述

普通话读音信息表，只读。

# 表结构

| 列名        | 内容               | 数据说明                     | 内容说明                                             |
|-----------|------------------|--------------------------|--------------------------------------------------|
| `id`      | 🔍 自增主键          | 自动生成列                    | 批量导入，插入顺序特意符合`ORDER BY zhuyin, tone, hanzi`的顺序   |
| `hanzi`   | 📖 汉字            |                          | 不区分简繁体                                           |
| `pinyin`  | 📢 汉语拼音（无音调）     |                          | 标注拼音程序`HanLP`生成的，拼音为键盘输入版本                       |
| `zhuyin`  | 📢 汉语注音（无音调）     |                          | 转换程序生成的，所以内容和上面的pinyin一一对应                       |
| `tone`    | 📢 数字音调          | 数字`0~4`                  | 所以轻声被标做`0`                                       |
| `info`    | 📖 综合信息          | 自动生成列<br/>（一般使用的就是合成的信息） | 直接拼`hanzi + " " + pinyin + tone`<br/>例如：`汉 han4` |
| `in_3500` | 📊 是否在大陆3500常用字里 | 数字`0`或`1`                | 因为多音字，所以`1`的数量要`GROUP BY hanzi`才是3500            |
| `in_4800` | 📊 是否在台湾4800常用字里 | 数字`0`或`1`                | 实际是4808个汉字，数量计算同上                                |

# 数据限定

- 拼音和注音一一对应。
- 汉字和拼音组合在一起为一个存储单位。同汉字的不同读音被视作不同的条目。
- 所以数据库使用的是`UNIQUE (hanzi, pinyin, zhuyin, tone)`作为唯一键，info也是唯一的，可以完全代表这个条目。
